# CSCO — 2025-11 → 2026-05 case study (the dog that didn't bark)

*Written 2026-05-29. Source data: `db/trading.db` (ohlcv_bars, indicator_snapshots,
signal_log, walk_forward_results, trade_log, order_decisions, ensemble_weight_history,
news_cache, universe_assets). No live fills exist for CSCO — this is a
**model-behavior** case study, not a trade case study. There is no entry, no exit,
no realised P&L. The subject is why the system **declined to participate** in a
+55.9% move.*

## TL;DR

Over the trailing 6 months CSCO returned **+55.9% vs SPY +11.0%** (+91.6% vs +29.9%
over 12 months) — one of the strongest risk-adjusted outperformers in the universe.
The system's stance on it was, and is, **persistently bearish**: the most recent
walk-forward run (2026-05-24) produced **all-zero folds** (Sharpe / drawdown /
win-rate = 0 across all 5), and recent live signals are stuck at **HOLD**.

The explanation is not a paradox. **The system is bearish on CSCO *because* CSCO is
winning, via the same causal channel that produced the outperformance.** A relentless
uptrend keeps the stock structurally overbought; XGBoost — whose target is the sign
of the **5-bar (~1 week) forward return** — has correctly learned "overbought →
short-term mean-revert" and reads the trend's own fingerprint as a sell trigger. The
LSTM (60-bar window, momentum-aware) reads the trend correctly and is **pinned at
+1.0**. The two models flatly contradict each other; the ensemble averages them to
near-neutral.

**Crucially, this cost opportunity, not money.** The system is long-only
(`allow_short_selling=False`), so every bearish CSCO signal is a no-op — there are
**zero trades** in `trade_log`. This is a model-*design* failure (short-horizon
mean-reversion opting out of a trend), not a bad prediction that lost capital. It is
the cleanest, most extreme instance of the documented universe-wide **SELL bias**,
and it is the strongest single motivating example for the staged **Option B** fix
(lengthen XGBoost's `_FORWARD_BARS` from 5 → ~20).

- **LSTM is saturated bullish** (+0.99 to +1.00) — it correctly reads the trend.
- **XGBoost is firmly bearish** (−0.43 to −0.98) — overbought → mean-revert → sell.
- The ensemble (latest weights LSTM **0.50** / XGB **0.25** / FinBERT **0.25**, with

push the ensemble negative, but it's strong enough to keep it below the buy gate. The
system is **blocked from buying** a stock its own trend model loves.

## 4. Why the Walk-Forward page showed all zeros

The 2026-05-24 WF run (`run_id 8ba654a2…`) for CSCO:

|---|---|---|---|---|---|
| 0 | 11 | 0.0 | 0.0 | 0.0 | suppressed: test window precedes news cutoff (2026-03-16) |

| 2 | 11 | 0.0 | 0.0 | 0.0 | suppressed: precedes news cutoff |
| 3 | 7 | 0.0 | 0.0 | 0.0 | — |
| 4 | 0 | 0.0 | 0.0 | 0.0 | — |

Two things to disentangle:

1. **The zeros in Sharpe/drawdown/win-rate are the SELL-bias, not missing data.**
   `n_signals` counts **passed-gate signals** (`models/walk_forward.py:266`) —
   including SELLs. Under long-only, a SELL-from-flat is a no-op, so no position is
   ever opened, every bar's return is 0, and `returns.std() == 0` trips the
   early-return-all-zeros branch in `_compute_fold_performance`
   (`models/walk_forward.py:741`). Zero trades → **empty `trade_log`** for the run.
   The folds *ran with full data*; the signals were SELLs the system can't act on.

2. **The `sentiment_note` "suppressed … precedes news cutoff" is the only place
   "new to universe" bites.** CSCO's earliest cached news is 2026-03-16, so folds
   whose test window predates that have FinBERT suppressed. This zeros out *FinBERT's
   contribution*, not the performance metrics.

**This is not CSCO-specific.** The same 2026-05-24 run produced all-zero folds for
**8 symbols**, every one with passed-gate signals (so none were data-starved): MSFT
(80 signals), ETN (59), ARM (56), TXN (45), **CSCO (36)**, GLW (30), NOK (24), MCHP
(19). **MSFT is a core watchlist mega-cap** — its presence definitively rules out
"new to universe" as the cause. All 8 are overbought tech/semis: the exact cluster
the SELL-bias diagnosis in CLAUDE.md describes.

## 5. Root cause — short-horizon mean-reversion in a trend

XGBoost's label is the **sign of the 5-bar forward return** (`models/xgboost_model.py:40,102`:
`_FORWARD_BARS = 5`). On a ~1-week horizon, an overbought reading often *is* followed
by a small pullback — so XGBoost's rule is empirically valid in a choppy/mean-reverting
market. But in a relentless uptrend the pullbacks are shallow and the trend resumes;
over the *whole* run the short-horizon bearish calls are swamped by the trend.

The causal loop, stated plainly:

```
strong uptrend  ──►  persistent overbought (high RSI, above-BB, above-EMA)
       │                          │
       │                          ▼
       │            XGBoost: "mean-revert" ──► SELL/bearish score
       ▼                          │
   +55.9% return                  ▼
   (LSTM: BUY)        ensemble dragged below buy gate ──► HOLD / no participation
```

The very feature values that scream "outperformer" to a human and to the LSTM are
the ones XGBoost was trained to fade. **XGBoost's edge (catching short-term
reversals) is its blind spot in a strong trend.**

## 6. Skill vs luck — what this episode actually cost

| Dimension | Read |
|---|---|
| Did the system lose money on CSCO? | **No.** Long-only + bearish = no-op. Zero trades, zero realised loss. |
| Did the system miss a winner? | **Yes.** It declined to participate in a +55.9% / 6-month move its LSTM rated +1.0. |
| Is this a prediction error or a design error? | **Design.** A 5-bar mean-reversion target structurally opts out of trends. The LSTM got it right; the ensemble averaged the right answer away. |
| Counterfactual upside | **Not quantifiable from realised data** (no fills). Directional only: a 20-bar XGBoost horizon would very likely flip XGBoost bullish on CSCO and let the ensemble clear the gate. |

Net read: **a system "miss" by design, not by luck.** This is the mirror image of the
AXTI case study (a system *success* that left money on the table by design). In both,
the lever is model/config design, not "let the model pick differently."

One nuance worth recording: when CSCO first entered the universe (2026-04-14 → 04-17),
`order_decisions` shows it *did* generate **BUY** signals (several `DRY_RUN`, one
`REJECTED` on position-size). So the bearish lock-in is not permanent — it tracks the
overbought regime. As the trend matured and RSI/BB readings climbed, XGBoost's
mean-reversion view took over and the ensemble settled into HOLD/SELL. That regime
dependence is consistent with the root cause in §5.

## 7. Implications

### 7a. Option B (lengthen XGBoost horizon) — **strongest single motivating case**

CSCO is the cleanest evidence for the staged **Option B** fix from the "Pervasive
SELL bias" entry in CLAUDE.md: change XGBoost's `_FORWARD_BARS` from 5 → ~20 so the
target captures **trend continuation** instead of short-term reversal. A 20-bar
forward return on CSCO over this window is overwhelmingly positive — XGBoost would
very likely flip bullish, the LSTM/XGBoost contradiction would resolve in the
*correct* direction, and the ensemble would clear the buy gate. Per CLAUDE.md, Option
B should only be pulled on **realised-P&L evidence (Phase B `source='live'` rows)**,
not on BUY:SELL ratios — but CSCO sharpens the hypothesis: the SELL bias is a
horizon-mismatch artifact, observable as a clean LSTM-vs-XGBoost split on strong
trenders.

### 7b. Reinforces "SELL bias is alpha-or-artifact, decide with live P&L"

The 2026-05-13 note in CLAUDE.md cautions against treating the SELL bias as a bug on
count-based grounds — XGBoost may be *correctly* bearish on names that subsequently
fall. CSCO is the counter-example that shows the bias can also be **flatly wrong** on
the universe's biggest winner. It doesn't settle the alpha-vs-artifact question (no
fills, no realised P&L), but it raises the stakes: the cost of the bias isn't only
"shorts we didn't take that would've worked" — it's also "trends we didn't ride."
This belongs in the evidence set when Phase B live data is used to make the Option B
call.

### 7c. Cross-reference to the LSTM↔indicator disagreement gate idea

The existing "LSTM ↔ MACD direction-disagreement gate" enhancement targets the
*opposite* failure (LSTM bullish + bearish indicator → block a bad BUY). CSCO is the
inverse: LSTM bullish + XGBoost bearish → the system *should* lean toward the LSTM on
a confirmed trend. Any future "model-disagreement" logic must be **regime-aware** —
in TRENDING, side with the trend model; in MEAN_REVERTING, the reversion model has
the better claim. A symmetric disagreement gate that always defers to caution would
make the CSCO miss *worse*.
